Hands-on Transcend tool


Transcend is NXP’s Managed File Transfer. It is widely used by R&D, Applications Engineering and Marketing teams to exchange information. While primarily focused on the Linux environment, clients are also available for Windows, Solaris and the Web. In production for many years, users tend to find Transcend more convenient than ftp, scp, or rsync solutions for data exchange and around 20,000 transfers per month are performed using the tool. Secure data repositories are distributed world-wide to optimize data posting and retrieval.

When the user transcends data, the data is securely posted to the repository. A nine character keyword is generated for data retrieval. Two forms of transfer are supported: general transfer where anyone who has the keyword is able to retrieve the data and secure transfer where the keyword is locked to the sender and targeted recipient email addresses.

A Transcend repository is available in each NXP site as distributed in the following table. Regional repositories are used improve Transcend user responsiveness. The 9th character of the keyword indicates the Repository where Transcend data is stored.

Data Repository Keyword Extension Example
Austin a abcd1234a
Eindhoven i abcd1234i
Munich m abcd1234m
Noida n abcd1234n
Bangalore b abcd1234b
External e abcd1234e

By default, each transfer is available during 3 days. After this 3 days a ‘get’ attempt using the keyword will return an error indicating that the data is no longer available. A special Transcend option allows to extend a keyword’s lifetime for up to 3 additional days.

The Transcend family of tools are under Freescale active development since 1994, helping in the engineering flow. Now Transcend is part of the NXP portfolio of tools. Transcend is available on Linux, Solaris, Windows and others platforms using TransWeb.

Traditionally, Transcend is the command line tool called trans available worldwide on the xFSL and NXDI data centers. A typical example of Transcend usage is a team with members living on different regions of the globe transferring files and using keywords to retrieve. The command line version has the best performance and availability. The tool is self-documented, we recommend it as preferable way of use and integrate Transcend in your projects.

Here on this hands-on guide is presented a short list of commands which exemplifying the typical workflow to integrate Transcend in your daily job.

Simply put data

You can use -p for put. 

$ trans –p random_file.txt

The -p option for put will works for directories too. 

$ trans –p random_dir

You also can use -p for put, notify a person’ email (Secure Keyword). 

$ trans –p random_file.txt john.doe@nxp.com

Retrieving (get) data

To retrieve a given keyword, use -g for get 

$ trans –g xqap6925a

Retrieving keyword information

You can use -i for general information, useful to know file expiration. 

$ trans –i xqap6925a

Retrieving keyword history

Use -h for history information, useful to know who retrieved the data and when. 

$ trans –h xqap6925a

Please note this command will save a file called xqap6925a.history in the relative PATH you are.

Avoid the keyword expiration by extending it

To avoid keyword expiration, call with option -e passing a number between 1-3 to extend file expiration from 1-3 extra days. 

$ trans -e xqap6925a 3

Lock/Unlock keywords

Transcend allow Lock/unlock data to prevent/allow data retrieval. The lock is done per file keyword, and can be performed soon after the upload toa repository.

To lock a given keyword, run: 

$ trans -L xqap6925a

then to unlock the data and allow data retrieval, run: 

trans -U xqap6925a

Get help about an option available

If you need to get help about an especific option, use -H follow by the option you are looking for. 

$ trans –H -e

Others tools

If you are looking for an option beyond the UNIX terminal and want to use from Windows for example, we recommend TransWeb: the Transcend based web application used to transfer files within NXP and between NXP and external groups. TransWeb offers the ability to send and receive files based on keywords but using a convenient web interface.

Visit http://transweb.freescale.net.
