What is Transcend?

Transcend (Transfer/Send Data) is the NXP's Managed File Transfer solution. It is widely used by R&D, Applications Engineering and Marketing teams to exchange information. While primarily focused on the Linux environment, clients are also available for Windows, Solaris and the Web. In production for many years, users tend to find transcend more convenient than ftp, scp, or rsync solutions for data exchange and around 20,000 transfers per month are performed using the tool. Secure data repositories are distributed world-wide to optimize data posting and retrieval.

When the user transcends data, the data is securely posted to the repository. A nine character keyword is generated for data retrieval. Two forms of transfer are supported: general transfer where anyone who has the keyword is able to retrieve the data and secure transfer where the keyword is locked to the sender and targeted recipient email addresses.

Transcend offers a higher level of security and control. Transcend uses secure socket connection (OpenSSL) to perform serial and parallel file transfer offering end-to-end encryption. Others features include notification of successful file transfers, auditability, and worldwide visibility.

Transcend tool have been widely used for data transfer between the worldwide sites, and is now available for whole NXP company. Transcend was first released for production use in July of 1994, and have been under continuos development from 22 years, currently being used by around 5000 users.

Release Notes

We are pleased to announce the production release of Transcend version 9.3.1 and TransWeb version 3.5.3.

The purpose of this maintenance release is enable Transcend and TransWeb to officially support the New NXP. Specific changes for this release include:

  • Adoption of the New NXP Active Directory for transcend authentication
  • Addition of two new repositories:
    • Eindhoven: trans -p -eind
    • Bangalore: trans -p -blr
  • Overall TransWeb UI face-lift
  • TransWeb login works with FSL CoreID (OneIT), NXP WBI ID or NXP e-mail. The login/authentication is using the NXP Active Directory with fall-back to the FSL LDAP in case of unsynchronized passwords
  • New Person Lookup from TransWeb also using the NXP Central Directory
  • Transfer as customer@freescale.com was deprecated.

The Transcend release has been uploaded to the Tools Repository with package fs-transcend-/9.3.1. The Transcend software can also be downloaded from the web site where packages for RHEL 4, 5, 6, Solaris, Ubuntu, Fedora and Windows are available. It is also available on xFSL datacenter as /usr/fsl/bin/trans and on oNXP datacenters as /nxdi_env/bin/trans.

IMPORTANT NOTE direct data transfers between Noida and Bangalore are currently blocked for legal reasons. Data transfers must be relayed through a repository outside of India.
