We are pleased to announce the production release of Transcend version 9.3.1 and TransWeb version 3.5.3.

The purpose of this maintenance release is enable Transcend and TransWeb to officially support the New NXP. Specific changes for this release include:

  • Adoption of the New NXP Active Directory for transcend authentication
  • Addition of two new repositories:
    • Eindhoven: trans -p -eind
    • Bangalore: trans -p -blr
  • Overall TransWeb UI face-lift
  • TransWeb login works with FSL CoreID (OneIT), NXP WBI ID or NXP e-mail. The login/authentication is using the NXP Active Directory with fall-back to the FSL LDAP in case of unsynchronized passwords
  • New Person Lookup from TransWeb also using the NXP Central Directory
  • Transfer as customer@freescale.com was deprecated.

The Transcend release has been uploaded to the Tools Repository with package fs-transcend-/9.3.1. The Transcend software can also be downloaded from the web site where packages for RHEL 4, 5, 6, Solaris, Ubuntu, Fedora and Windows are available. It is also available on xFSL datacenter as /usr/fsl/bin/trans and on oNXP datacenters as /nxdi_env/bin/trans.

IMPORTANT NOTE direct data transfers between Noida and Bangalore are currently blocked for legal reasons. Data transfers must be relayed through a repository outside of India.
